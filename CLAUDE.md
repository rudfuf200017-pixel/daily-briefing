# 가시제거연구소 데일리 브리핑 — 원격 루틴 작업 지시서

## 실행 방법

```
python main.py
```

이 명령 하나만 실행한다. 추가 웹 검색, 큐레이션, 수동 편집 절대 금지.

## 매일 자동 실행 규칙

- 실행 시각: 매일 09:00 KST
- Claude 원격 루틴(스케줄)이 `python main.py`를 호출한다.
- 실행 전 추가 조사나 판단 없이 즉시 실행한다.

## 수집 로직 — 섹션별 2단계 폴백

각 섹션(시장뉴스 / 네이버블로그 / 유튜브)은 독립적으로 아래 순서를 따른다.

### 1순위 (기본)
- 섹션 내 모든 RSS의 KST 기준 최근 24시간 내 신규 항목 합산
- 발행시각 내림차순 정렬
- 시장뉴스: 최대 3건 / 블로그·유튜브: 최대 5건

### 2순위 폴백 (1순위 결과 0건일 때만 발동)
- 시장뉴스: 섹션 전체 RSS에서 발행시각 최근 3건 (기간 제한 없음)
- 블로그/유튜브: 소스(브랜드/채널)별 최신 1건씩, 합산 최대 5건

**빈 페이지 절대 금지.** 신규 항목이 없어도 폴백으로 최근 N건을 반드시 채운다.

## 섹션 헤더 라벨 규칙

임원이 신규/참고를 한눈에 구분할 수 있도록 heading_2에 라벨을 포함한다.

| 상황 | 라벨 예시 |
|------|-----------|
| 24h 신규 있음 | `1. 시장 핵심 뉴스 🆕 24시간 내 신규 3건` |
| 폴백 발동 | `1. 시장 핵심 뉴스 📚 24h 신규 없음 — 최근 3건 참고` |

## RSS 실패 처리

- 개별 RSS URL 접속 실패 → 해당 소스만 건너뛰고 나머지 소스로 계속 진행
- 섹션 전체 실패 → 폴백 발동 (폴백 결과도 0건이면 섹션 헤더만 남음)
- 노션 페이지 생성 실패 → 오류 메시지 출력 후 종료 (exit code 1)

## 콘솔 출력 항목

실행 시 아래 정보를 콘솔에 출력한다:

1. 시작 시각 (KST)
2. 소스별 수신 건수 또는 실패 이유
3. 섹션별 1순위 성공 / 폴백 발동 여부 + 건수
4. 노션 페이지 URL

## 금지 사항

- AI API 호출 금지 (LLM 요약·번역·재작성 등 일체)
- 노션 API는 단순 페이지 생성 용도로만 허용
- RSS 원본 데이터 가공 금지 (description 앞 110자 절단 제외)
- 수치·고유명사·브랜드명 변경 금지
- 추가 웹 검색, 피드 추가, 큐레이션 금지

## 환경변수

| 변수명 | 설명 |
|--------|------|
| `NOTION_API_KEY` | Notion Integration Secret |
| `NOTION_PARENT_PAGE_ID` | 브리핑 페이지가 생성될 부모 페이지 ID |

값은 Claude 클라우드 환경 설정(Secrets)에 등록. `.env` 파일에도 저장 가능 (gitignore됨).
